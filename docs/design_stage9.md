# 阶段 9：测试与验证（规划）

目标：定义最小可验证方案（含无设备 stub）、单元/集成测试思路，覆盖核心逻辑（模型、管线、服务）。仅文档。

## 测试策略
- **无设备环境**：
  - 提供 `TapeDevice` stub：返回可控数据/错误；用于 CI 骨架与 UI 调试。
  - Mock `TapeEnumerator`：返回虚拟设备列表。
- **模型测试**：
  - `ltfs_index`：序列化/反序列化一致性；排序与选择逻辑；xattr 读写；时间格式；非法字符替换。
  - `ltfs_label`：参数校验（blocksize/partition 数量/extra partition），与 mkltfs 参数映射。
- **管线测试**：
  - `FileDataProvider`：
    - 小文件缓存阈值、缓存容量触发。
    - 背压阈值：写入/读取速度差异导致的阻塞/恢复。
    - requireSignal 模式：不调用信号则不会推进。
    - 取消/完成：确保线程退出且资源释放。
  - 限速：在模拟时间推进下验证速率控制。
- **服务测试**：
  - `LtfsService`（使用 stub TapeDevice）：
    - 加载 label/index 成功/失败路径。
    - 写入流程：数据管线 -> 写块/文件标记；索引周期写入触发。
    - 容量刷新调用频率；状态/回调顺序。
- **CLI 测试**：
  - 参数解析路由；未实现命令的提示。

## 工具与框架
- 可选：Qt Test 或 GoogleTest（根据后续选型）。
- CI：
  - 构建矩阵（Linux/Windows 至少，macOS 可选）；无设备模式运行全部测试。

## 验收点
- 在无设备 stub 下，可编译并运行全部单元测试。
- 关键行为（选择逻辑、背压、限速、索引读写）有自动化覆盖。
